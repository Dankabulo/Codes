Stockage de fichiers et accès avec Xamarin. AndroidFile Storage and Access with Xamarin.Android

  • 23/07/2018
  • 7 minutes de lecture

Dans cet article

  1. Stockage interne et externe
  2. Utilisation du stockage interne
  3. Utilisation des – applications auxiliaires du système de fichiers Xamarin. Essentials
  4. Masquage des fichiers du MediaStore
  5. Liens associés

Une exigence courante pour les applications Android consiste à manipuler des fichiers pour – enregistrer des images, télécharger des documents ou exporter des données à partager avec d’autres programmes.A common requirement for Android apps is to manipulate files – saving pictures, downloading documents, or exporting data to share with other programs. Android (basé sur Linux) le prend en charge en fournissant de l’espace pour le stockage des fichiers.Android (which is based on Linux) supports this by providing space for file storage. Android regroupe le système de fichiers en deux types de stockage différents :Android groups the filesystem into two different types of storage:

  • Stockage interne – Il s’agit d’une partie du système de fichiers accessible uniquement par l’application ou le système d’exploitation.Internal Storage – this is a portion of the file system that can be accessed only by the application or the operating system.
  • Stockage externe – Il s’agit d’une partition pour le stockage des fichiers accessibles par toutes les applications, l’utilisateur et éventuellement d’autres appareils.External Storage – this is a partition for the storage of files that is accessible by all apps, the user, and possibly other devices. Sur certains appareils, le stockage externe peut être amovible (par exemple, une carte SD).On some devices, external storage may be removable (such as an SD card).

Ces regroupements sont conceptuels uniquement et ne font pas nécessairement référence à une partition ou un répertoire unique sur l’appareil.These groupings are conceptual only, and don't necessarily refer to a single partition or directory on the device. Un appareil Android fournira toujours une partition pour le stockage interne et le stockage externe.An Android device will always provide partition for internal storage and external storage. Il est possible que certains appareils puissent avoir plusieurs partitions qui sont considérées comme du stockage externe.It is possible that certain devices may have multiple partitions that are considered to be external storage. Quelle que soit la partition, les API pour la lecture, l’écriture ou la création de fichiers sont les mêmes.Regardless of the partition the APIs for reading, writing, or creating files is the same. Il existe deux ensembles d’API qui peuvent être utilisés par une application Xamarin. Android pour accéder aux fichiers :There are two sets of APIs that a Xamarin.Android application may use for file access:

  1. API .net (fournies par mono et encapsulées par Xamarin. Android) – Celles-ci incluent les applications auxiliaires de système de fichiers fournies par Xamarin. Essentials.The .NET APIs (provided by Mono and wrapped by Xamarin.Android) – These includes the file system helpers provided by Xamarin.Essentials. Les API .NET offrent la meilleure compatibilité multiplateforme et, par conséquent, ce guide se concentre sur ces API.The .NET APIs provide the best cross-platform compatibility and as such the focus of this guide will be on these APIs.
  2. Les API d’accès au fichier Java natif (fournies par Java et encapsulées par Xamarin. Android) – Java fournit ses propres API pour la lecture et l’écriture de fichiers.The native Java file access APIs (provided by Java and wrapped by Xamarin.Android) – Java provides its own APIs for reading and writing files. Il s’agit d’une alternative entièrement acceptable pour les API .NET, mais qui sont spécifiques à Android et ne conviennent pas aux applications qui sont destinées à être multiplateformes.These are a completely acceptable alternative to the .NET APIs, but are specific to Android and are not suitable for apps that are intended to be cross-platform.

La lecture et l’écriture dans des fichiers sont presque identiques dans Xamarin. Android, comme c’est le cas pour toute autre application .NET.Reading and writing to files is almost identical in Xamarin.Android as it is to any other .NET application. L’application Xamarin. Android détermine le chemin d’accès au fichier qui sera manipulé, puis utilise les idiomes .NET standard pour l’accès aux fichiers.The Xamarin.Android app determines the path to the file that will be manipulated, then uses standard .NET idioms for file access. Étant donné que les chemins d’accès réels au stockage interne et externe peuvent varier d’un appareil à l’appareil ou d’une version Android à une version Android, il n’est pas recommandé de coder en dur le chemin d’accès aux fichiers.Because the actual paths to internal and external storage may vary from device to device or from Android version to Android version, it is not recommended to hard code the path to the files. Utilisez plutôt les API Xamarin. Android pour déterminer le chemin d’accès aux fichiers.Instead, use the Xamarin.Android APIs to determine the path to files. Ainsi, les API .NET pour la lecture et l’écriture de fichiers exposent les API Android natives qui vous aideront à déterminer le chemin d’accès aux fichiers sur le stockage interne et externe.That way, the .NET APIs for reading and writing files exposes the native Android APIs that will help with determining the path to files on internal and external storage.

Avant de discuter des API impliquées dans l’accès aux fichiers, il est important de comprendre certains détails relatifs au stockage interne et externe.Before discussing the APIs involved with file access, it is important to understand some of the details surrounding internal and external storage. Ce sujet sera abordé dans la section suivante.This will be discussed in the next section.

Stockage interne et externeInternal vs external storage

Conceptuellement, le stockage interne et le stockage externe sont très similaires – à ceux où une application Xamarin. Android peut enregistrer des fichiers.Conceptually, internal storage and external storage are very similar – they are both places at which a Xamarin.Android app may save files. Cette similarité peut prêter à confusion pour les développeurs qui ne sont pas familiarisés avec Android, car il n’est pas évident qu’une application doit utiliser le stockage interne et le stockage externe.This similarity may be confusing for developers who are not familiar with Android as it is not clear when an app should use internal storage vs external storage.

Le stockage interne fait référence à la mémoire non volatile que Android alloue au système d’exploitation, fichiers APK et pour les applications individuelles.Internal storage refers to the non-volatile memory that Android allocates to the operating system, APKs, and for individual apps. Cet espace n’est pas accessible, sauf par le système d’exploitation ou les applications.This space is not accessible except by the operating system or apps. Android allouera un répertoire dans la partition de stockage interne pour chaque application.Android will allocate a directory in the internal storage partition for each app. Lorsque l’application est désinstallée, tous les fichiers qui sont conservés sur le stockage interne dans ce répertoire sont également supprimés.When the app is uninstalled, all the files that are kept on internal storage in that directory will also be deleted. Le stockage interne est mieux adapté aux fichiers qui ne sont accessibles qu’à l’application et qui ne sont pas partagés avec d’autres applications ou qui n’ont pas une valeur minimale une fois l’application désinstallée.Internal storage is best suited for files that are only accessible to the app and that will not be shared with other apps or will have very little value once the app is uninstalled. Sur Android 6,0 ou version ultérieure, les fichiers sur le stockage interne peuvent être sauvegardés automatiquement par Google à l’aide de la fonctionnalité de sauvegarde automatique dans Android 6,0.On Android 6.0 or higher, files on internal storage may be automatically backed up by Google using the Auto Backup feature in Android 6.0. Le stockage interne présente les inconvénients suivants :Internal storage has the following disadvantages:

  • Les fichiers ne peuvent pas être partagés.Files cannot be shared.
  • Les fichiers seront supprimés lors de la désinstallation de l’application.Files will be deleted when the app is uninstalled.
  • L’espace disponible sur le stockage interne peut être limité.The space available on internal storage maybe limited.

Le stockage externe fait référence au stockage de fichiers qui n’est pas un stockage interne et n’est pas exclusivement accessible à une application.External storage refers to file storage that is not internal storage and not exclusively accessible to an app. L’objectif principal du stockage externe est de fournir un emplacement pour placer les fichiers destinés à être partagés entre les applications ou qui sont trop volumineux pour tenir sur le stockage interne.The primary purpose of external storage is to provide a place to put files that are meant to be shared between apps or that are too large to fit on the internal storage. L’avantage du stockage externe est qu’il a généralement plus d’espace pour les fichiers que le stockage interne.The advantage of external storage is that it typically has much more space for files than internal storage. Toutefois, le stockage externe n’est pas toujours présent sur un appareil et peut nécessiter une autorisation spéciale de la part de l’utilisateur pour y accéder.However, external storage is not always guaranteed to be present on a device and may require special permission from the user to access it.

Notes

Pour les appareils qui prennent en charge plusieurs utilisateurs, Android fournit à chaque utilisateur leur propre répertoire sur le stockage interne et externe.For devices that support multiple users, Android will provide each user their own directory on both internal and external storage. Ce répertoire n’est pas accessible aux autres utilisateurs sur l’appareil.This directory is inaccessible to other users on the device. Cette séparation est invisible aux applications tant qu’elles ne sont pas codées en dur dans des fichiers sur un stockage interne ou externe.This separation is invisible to apps as long as they do not hardcode paths to files on internal or external storage.

En règle générale, les applications Xamarin. Android doivent préférer enregistrer leurs fichiers sur le stockage interne lorsque cela est raisonnable et s’appuyer sur un stockage externe lorsque les fichiers doivent être partagés avec d’autres applications, sont très volumineux ou doivent être conservés même si l’application est désinstallée.As a rule of thumb, Xamarin.Android apps should prefer saving their files on internal storage when it is reasonable, and rely on external storage when files need to be shared with other apps, are very large, or should be retained even if the app is uninstalled. Par exemple, un fichier de configuration est mieux adapté à un stockage interne, car il n’a aucune importance, sauf pour l’application qui le crée.For example, a configuration file is best suited for a internal storage as it has no importance except to the app that creates it. En revanche, les photos sont un bon candidat pour le stockage externe.In contrast, photos are a good candidate for external storage. Ils peuvent être très volumineux et, dans de nombreux cas, l’utilisateur peut souhaiter le partager ou y accéder même si l’application est désinstallée.They can be very large and in many cases the user may want to share them or access them even if the app is uninstalled.

Ce guide se concentre sur le stockage interne.This guide will focus on internal storage. Pour plus d’informations sur l’utilisation du stockage externe dans une application Xamarin. Android, consultez le guide stockage externe .Please see the guide External storage for details on using external storage in a Xamarin.Android application.

Utilisation du stockage interneWorking with internal storage

Le répertoire de stockage interne pour une application est déterminé par le système d’exploitation et est exposé aux applications Android par la Android.Content.Context.FilesDir propriété.The internal storage directory for an application is determined by the operating system, and is exposed to Android apps by the Android.Content.Context.FilesDir property. Cela renverra un Java.IO.File objet représentant le répertoire qu’Android a exclusivement dédié à l’application.This will return a Java.IO.File object representing the directory that Android has dedicated exclusively for the app. Par exemple, une application avec le nom de package com. CompanyName le répertoire de stockage interne peut être :For example, an app with the package name com.companyname the internal storage directory might be:

Bash
/data/user/0/com.companyname/files

Ce document fait référence au répertoire de stockage interne en tant que _ _ stockage interne_.This document will refer to the internal storage directory as INTERNAL_STORAGE.

Important

Le chemin d’accès exact au répertoire de stockage interne peut varier d’un appareil à l’autre et entre les versions d’Android.The exact path to the internal storage directory can vary from device to device and between versions of Android. Pour cette raison, les applications ne doivent pas coder en dur le chemin d’accès au répertoire de stockage des fichiers internes et utiliser à la place les API Xamarin. Android, telles que System.Environment.GetFolderPath() .Because of this, apps must not hard code the path to the internal files storage directory, and instead use the Xamarin.Android APIs, such as System.Environment.GetFolderPath().

Pour optimiser le partage de code, les applications Xamarin. Android (ou les applications Xamarin. Forms ciblant Xamarin. Android) doivent utiliser la System.Environment.GetFolderPath() méthode.To maximize code sharing, Xamarin.Android apps (or Xamarin.Forms apps targeting Xamarin.Android) should use the System.Environment.GetFolderPath() method. Dans Xamarin. Android, cette méthode retourne une chaîne pour un répertoire qui est le même emplacement que Android.Content.Context.FilesDir .In Xamarin.Android, this method will return a string for a directory that is the same location as Android.Content.Context.FilesDir. Cette méthode prend une énumération, System.Environment.SpecialFolder , qui est utilisée pour identifier un jeu de constantes énumérées qui représentent les chemins d’accès des dossiers spéciaux utilisés par le système d’exploitation.This method takes an enum, System.Environment.SpecialFolder, which is used to identify a set of enumerated constants that represent the paths of special folders used by the operating system. Toutes les valeurs ne System.Environment.SpecialFolder sont pas mappées à un répertoire valide sur Xamarin. Android.Not all of the System.Environment.SpecialFolder values will map to a valid directory on Xamarin.Android. Lecture ou écriture dans des fichiers sur le stockage interneReading or Writing to files on internal storage

Toutes les API C# pour l’écriture dans un fichier sont suffisantes ; le chemin d’accès au fichier qui se trouve dans le répertoire alloué à l’application est tout ce qui est nécessaire.Any of the C# APIs for writing to a file are sufficient; all that is necessary is to get the path to the file that is in the directory allocated to the application. Il est vivement recommandé d’utiliser les versions Async des API .NET pour réduire les problèmes qui peuvent être associés à l’accès aux fichiers bloquant le thread principal.It is strongly recommended that the async versions of the .NET APIs are used to minimize any issues that may be associate with file access blocking the main thread.

Cet extrait de code est un exemple d’écriture d’un entier dans un fichier texte UTF-8 dans le répertoire de stockage interne d’une application :This code snippet is one example of writing an integer to a UTF-8 text file to the internal storage directory of an application:

C#
public async Task SaveCountAsync(int count)
{
    var backingFile = Path.Combine(System.Environment.GetFolderPath(System.Environment.SpecialFolder.Personal), "count.txt");
    using (var writer = File.CreateText(backingFile))
    {
        await writer.WriteLineAsync(count.ToString());
    }
}

L’extrait de code suivant fournit un moyen de lire une valeur entière qui a été stockée dans un fichier texte :The next code snippet provides one way to read an integer value that was stored in a text file:

C#
public async Task<int> ReadCountAsync()
{
    var backingFile = Path.Combine(System.Environment.GetFolderPath(System.Environment.SpecialFolder.Personal), "count.txt");

    if (backingFile == null || !File.Exists(backingFile))
    {
        return 0;
    }

    var count = 0;
    using (var reader = new StreamReader(backingFile, true))
    {
        string line;
        while ((line = await reader.ReadLineAsync()) != null)
        {
            if (int.TryParse(line, out var newcount))
            {
                count = newcount;
            }
        }
    }

    return count;
}

Utilisation des – applications auxiliaires du système de fichiers Xamarin. EssentialsUsing Xamarin.Essentials – File System Helpers

Xamarin. Essentials est un ensemble d’API permettant d’écrire du code compatible multiplateforme.Xamarin.Essentials is a set of APIs for writing cross-platform compatible code. Les applications auxiliaires du système de fichiers sont une classe qui contient une série d’applications auxiliaires permettant de simplifier la localisation des répertoires de cache et de données de l’application.The File System Helpers is a class that contains a series of helpers to simplify locating the application's cache and data directories. Cet extrait de code fournit un exemple de la façon de rechercher le répertoire de stockage interne et le répertoire de cache pour une application :This code snippet provides an example of how to find the internal storage directory and the cache directory for an app:

C#
// Get the path to a file on internal storage
var backingFile = Path.Combine(Xamarin.Essentials.FileSystem.AppDataDirectory, "count.txt");

// Get the path to a file in the cache directory
var cacheFile = Path.Combine(Xamarin.Essentials.FileSystem.CacheDirectory, "count.txt");

Masquage des fichiers du MediaStoreHiding files from the MediaStore

Le MediaStore est un composant Android qui collecte des métadonnées sur les fichiers multimédias (vidéos, musique, images) sur un appareil Android.The MediaStore is an Android component that collects meta data about media files (videos, music, images) on an Android device. Son objectif est de simplifier le partage de ces fichiers sur toutes les applications Android sur l’appareil.Its purpose is simplify the sharing of these files across all Android apps on the device.

Les fichiers privés ne s’affichent pas en tant que média partageable.Private files will not show up as shareable media. Par exemple, si une application enregistre une image dans son stockage externe privé, ce fichier n’est pas récupéré par le scanneur multimédia ( MediaStore ).For example, if an app saves a picture to its private external storage, then that file will not be picked up by the media scanner (MediaStore).

Les fichiers publics sont récupérés par MediaStore .Public files will be picked up by MediaStore. Répertoires qui ont un nom de fichier de zéro octet . Nomedias ne sera pas analysé par MediaStore .Directories that have a zero byte file name .NOMEDIA will not be scanned by MediaStore.
